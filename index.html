SnapSetup Help

Everything you need to know about converting projects into portable setup scripts.

1 What is SnapSetup?

SnapSetup scans a project folder in your browser and generates two self-contained scripts:

  • setup.ps1 — PowerShell script for Windows
  • setup.sh — Bash script for macOS / Linux

Running either script recreates the full folder structure and all files — no zip, no installer, just one script.

2 How to Use

  • Select folder — Click the drop zone or drag & drop a folder onto it.
  • Configure — Optionally set a project name, max file size, and exclude patterns.
  • Generate — Click the Generate Setup Files button.
  • Get output — Download or copy the generated scripts.

3 Options Explained

  • Project Name — Defaults to the folder name. Used as the root directory when scripts are run.
  • Max File Size — Files above this limit (in MB) are skipped. Default: 10 MB.
  • Exclude Patterns — One pattern per line. Files or folders matching these patterns are skipped.

4 Default Exclusions

These folders and files are always excluded:

  • Folders: node_modules, .git, __pycache__, .venv, venv, dist, build, .vs, .vscode, .idea, bin, obj
  • Files: *.pyc, *.pyo, *.log, *.tmp, *.bak, Thumbs.db, .DS_Store

5 Copy vs Download

Copy to Clipboard is recommended. Paste into a locally-created file to avoid the Windows “Mark of the Web” security warning that blocks downloaded scripts.

Download works too, but you may need to right-click the file → Properties → Unblock before running.

6 Running the Scripts

  • Windows: Open PowerShell, navigate to the script folder, then run .\setup.ps1
  • macOS/Linux: Open Terminal, run chmod +x setup.sh then ./setup.sh

7 Browser Requirements

SnapSetup uses the File System Access API to read folders. This requires Chrome or Edge (version 86+). Firefox and Safari are not supported.
